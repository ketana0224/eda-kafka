## 命名規約

CAF（Cloud Adoption Framework）推奨プレフィックスを使用する。

- **基本パターン**: `{prefix}-{env}-{project}`
  - `{env}` = 環境識別子（例: `ketana-ext2`）
  - `{project}` = プロジェクト名（例: `eda-kafka`）
- **例外**: コンテナーレジストリはハイフン不可のため英数字のみ（`cr{env}{project}`）

| リソース種別 | CAF プレフィックス | 備考 |
|---|---|---|
| リソースグループ | `rg-` | |
| コンテナーグループ | `ci-` | ACI はコンテナーグループ単位 |
| 仮想ネットワーク | `vnet-` | |
| サブネット | `snet-` | |
| ネットワークセキュリティグループ | `nsg-` | |
| コンテナーレジストリ | `cr` | ハイフン・アンダースコア不可、英数字のみ |

> CAF 公式参照: [Azure リソースの省略形](https://learn.microsoft.com/ja-jp/azure/cloud-adoption-framework/ready/azure-best-practices/resource-abbreviations)

## イメージ参照ドキュメント

> 参照先: [https://hub.docker.com/r/apache/kafka](https://hub.docker.com/r/apache/kafka)

> **注意**: `bitnami/kafka` は Docker Hub から完全撤退済み（count:0）。`apache/kafka` が公式の置き換えイメージ。

### apache/kafka の重要事項

| 項目 | 内容 |
|---|---|
| イメージ取得 | `docker pull apache/kafka:latest` |
| モード | KRaft（ZooKeeper 不要） |
| 環境変数プレフィックス | `KAFKA_`（bitnami の `KAFKA_CFG_` とは異なる） |
| `CLUSTER_ID` | KRaft モードで必須。未指定時のデフォルト: `5L6g3nShT-eMCtK--X86sw` |

**KRaft モード必須環境変数:**
```
CLUSTER_ID=5L6g3nShT-eMCtK--X86sw
KAFKA_NODE_ID=1
KAFKA_PROCESS_ROLES=broker,controller
KAFKA_LISTENERS=PLAINTEXT://:9092,CONTROLLER://:9093
KAFKA_ADVERTISED_LISTENERS=PLAINTEXT://<外部IP>:9092
KAFKA_CONTROLLER_QUORUM_VOTERS=1@localhost:9093
KAFKA_CONTROLLER_LISTENER_NAMES=CONTROLLER
KAFKA_LISTENER_SECURITY_PROTOCOL_MAP=CONTROLLER:PLAINTEXT,PLAINTEXT:PLAINTEXT
KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR=1
KAFKA_TRANSACTION_STATE_LOG_REPLICATION_FACTOR=1
KAFKA_TRANSACTION_STATE_LOG_MIN_ISR=1
```

### ACI 環境での Docker Hub アクセス

- **ACI パブリックIP 構成**: Docker Hub へ直接アクセス可能（`nginx:latest` プルで実証済み）
- **ACI + VNet統合**: Docker Hub への直接アクセス不可 → ACR経由が必要
- **bitnami/kafka の代替**: `apache/kafka:latest` を Docker Hub から直接プル可能

## Plan

### Phase 1 — 前提条件確認
1. サブスクリプション設定
   - `az account list` でサブスクリプション `677a9591-0639-4aa6-9923-d2d2f4dfc132` が存在するか確認する
   - 存在しない場合はテナントを指定して再ログインする:
     ```powershell
     az login --tenant 655bd66a-5001-4cb3-9aad-ce54a27d5d95
     ```
   - 再ログイン後も見つからない場合はアクセス権を確認して停止する
   - 確認後に設定:
     ```powershell
     az account set --subscription 677a9591-0639-4aa6-9923-d2d2f4dfc132
     az account show --query "{id:id, name:name}" -o table
     ```
   > スクリプト (`deploy-kafka-aci.ps1`) はこの確認ロジックを自動実行する
2. リソースプロバイダー登録確認
   - `Microsoft.ContainerInstance`
   - `Microsoft.Network`
3. リソースグループの確認・作成
   - `az group show` でリソースグループの存在を確認する
   - **存在しない場合**: `az group create` で作成する
   - **存在する場合**: 人間に確認を求め、`yes` なら削除して再作成、`no` なら既存のまま続行する
     ```powershell
     $confirm = Read-Host "リソースグループ '$resourceGroup' は既に存在します。削除して再作成しますか？ (yes/no)"
     if ($confirm -eq "yes") {
         az group delete --name $resourceGroup --yes
         # 削除完了まで待機（ResourceGroupBeingDeleted 回避）
         do {
             Start-Sleep -Seconds 10
             $state = az group show --name $resourceGroup --query properties.provisioningState -o tsv 2>$null
             Write-Host "削除待機中: $state"
         } while ($state)
         az group create --name $resourceGroup --location $location
     }
     ``` — ACI に Kafka コンテナをデプロイ
4. `az container create`（1回目）で `cg-kafka` を作成し、パブリックIPを取得する
   - イメージ: `apache/kafka:latest`（Docker Hub から直接プル）
   - ポート: `9092`（クライアント）, `9093`（KRaft コントローラー）
   - ネットワーク: パブリックIP（VNet なし）
   - CPU: 1コア, メモリ: 2GB
   - 環境変数（`KAFKA_ADVERTISED_LISTENERS` は除く）:
     ```
     CLUSTER_ID=5L6g3nShT-eMCtK--X86sw
     KAFKA_NODE_ID=1
     KAFKA_PROCESS_ROLES=broker,controller
     KAFKA_LISTENERS=PLAINTEXT://:9092,CONTROLLER://:9093
     KAFKA_CONTROLLER_QUORUM_VOTERS=1@localhost:9093
     KAFKA_CONTROLLER_LISTENER_NAMES=CONTROLLER
     KAFKA_LISTENER_SECURITY_PROTOCOL_MAP=CONTROLLER:PLAINTEXT,PLAINTEXT:PLAINTEXT
     KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR=1
     KAFKA_TRANSACTION_STATE_LOG_REPLICATION_FACTOR=1
     KAFKA_TRANSACTION_STATE_LOG_MIN_ISR=1
     ```
5. 取得したIPで `az container create`（2回目）を実行し、`KAFKA_ADVERTISED_LISTENERS` を確定IPで設定する
   ```
   KAFKA_ADVERTISED_LISTENERS=PLAINTEXT://<ACI-IP>:9092
   ```
6. `infra/config.psd1` に全パラメータを定義し、`infra/deploy-kafka-aci.ps1` から `Import-PowerShellDataFile` で読み込む

### Phase 3 — EntraID ロール割り当て
7. 実行ユーザーに対してリソースグループスコープで以下を付与
   - `Contributor`（ACI管理用）

## Verification

> **前提**: 以下のコマンドで設定を読み込む（値は [0.config.md](0.config.md) 参照）
> ```powershell
> $cfg = Import-PowerShellDataFile .\infra\config.psd1
> $ip  = az container show -n $cfg.ContainerGroup -g $cfg.ResourceGroup --query ipAddress.ip -o tsv
> ```

### Phase 1 — 前提条件
1. `az account show --query id -o tsv` でサブスクリプションID `$cfg.SubscriptionId` が表示されること
2. `az provider show -n Microsoft.ContainerInstance --query registrationState` が `Registered` であること
3. `az group show -n $cfg.ResourceGroup --query location` が `$cfg.Location` であること

### Phase 2 — Kafka コンテナ動作確認
4. `az container show -n $cfg.ContainerGroup -g $cfg.ResourceGroup --query instanceView.state` が `Running` であること
5. `az container logs -n $cfg.ContainerGroup -g $cfg.ResourceGroup` に KRaft での起動完了ログが出力されること
6. テストトピック作成・メッセージ送受信が正常に完了すること
   ```powershell
   docker run --rm apache/kafka `
     /opt/kafka/bin/kafka-topics.sh --create --topic test `
     --bootstrap-server "${ip}:9092"

   docker run --rm -i apache/kafka `
     /opt/kafka/bin/kafka-console-producer.sh --topic test `
     --bootstrap-server "${ip}:9092"

   docker run --rm apache/kafka `
     /opt/kafka/bin/kafka-console-consumer.sh --topic test `
     --from-beginning --bootstrap-server "${ip}:9092"
   ```

### Phase 3 — ロール
7. `az role assignment list --assignee <UPN> --scope /subscriptions/$($cfg.SubscriptionId)/resourceGroups/$($cfg.ResourceGroup)` に `Contributor` が含まれること
