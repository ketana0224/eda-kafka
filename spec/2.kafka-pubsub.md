## Plan

### Phase 1 — プロジェクト骨格
1. `src/pom.xml` 作成（`kafka-clients` 依存のみ、Spring Boot なし）
2. `src/main/resources/kafka.properties` 作成（BootstrapServers を外部設定で切替可能に）

### Phase 2 — AdminClient（Topic 操作）
3. `TopicAdmin.java`: AdminClient で Topic 作成（3パーティション）・一覧・削除を実装

### Phase 3 — Producer / Consumer 基本疎通
4. `SimpleProducer.java`: Key なしメッセージ送信
5. `SimpleConsumer.java`: 単一 Consumer Group で受信

### Phase 4 — Key あり/なし検証
6. `KeyedProducer.java`: Key 付きメッセージ送信
   - Key なし → 3パーティションにラウンドロビン分散
   - Key あり → 同一 Key は同一パーティションに集約

### Phase 5 — 複数 Consumer 検証
7. `MultiConsumer.java`
   - **同一 Group**: メッセージが Consumer 間で分散（各メッセージは1つの Consumer のみ受信）
   - **別 Group**: 各 Group が同一 Topic から全メッセージを受信

### Phase 6 — Verification 追記
8. 本ファイルに Verification セクションを追記

## Verification

> **前提**: Maven と JAVA_HOME を設定してから実行
> ```powershell
> $mvn = "C:\tools\maven\apache-maven-3.9.6\bin\mvn.cmd"
> $env:JAVA_HOME = "C:\Program Files\Java\jdk-20"
> ```

### Phase 2 — AdminClient（Topic 操作）

1. Topic 作成
   ```powershell
   & $mvn exec:java "-Dexec.mainClass=com.example.kafka.TopicAdmin" "-Dexec.args=create pubsub-test"
   ```
   → `Created topic: pubsub-test (partitions=3)` が出力されること

2. Topic 一覧
   ```powershell
   & $mvn exec:java "-Dexec.mainClass=com.example.kafka.TopicAdmin" "-Dexec.args=list"
   ```
   → `pubsub-test` が一覧に表示されること

3. Topic 削除
   ```powershell
   & $mvn exec:java "-Dexec.mainClass=com.example.kafka.TopicAdmin" "-Dexec.args=delete pubsub-test"
   ```
   → `Deleted topic: pubsub-test` が出力されること

### Phase 3 — Producer / Consumer 基本疎通

4. `SimpleProducer` — Key なし 10件送信
   ```powershell
   & $mvn exec:java "-Dexec.mainClass=com.example.kafka.SimpleProducer" "-Dexec.args=pubsub-test 10"
   ```
   → `Sent: value=message-N partition=X offset=Y` が10行出力されること

5. `SimpleConsumer` — group-a で 10件受信
   ```powershell
   & $mvn exec:java "-Dexec.mainClass=com.example.kafka.SimpleConsumer" "-Dexec.args=pubsub-test group-a 10"
   ```
   → `Done. Total received: 10` が出力されること

### Phase 4 — Key あり/なし検証

6. `KeyedProducer nokey` — Key なし送信
   ```powershell
   & $mvn exec:java "-Dexec.mainClass=com.example.kafka.KeyedProducer" "-Dexec.args=pubsub-test nokey"
   ```
   → Kafka 2.4+ のスティッキーパーティショナーにより、小バッチは同一パーティションに集中する
   （実測: 全メッセージが partition=1 に集中）

7. `KeyedProducer keyed` — Key あり送信
   ```powershell
   & $mvn exec:java "-Dexec.mainClass=com.example.kafka.KeyedProducer" "-Dexec.args=pubsub-test keyed"
   ```
   → 同一 Key が必ず同一パーティションに集約されること
   （実測: user-A → partition=0, user-B → partition=0, user-C → partition=2）

### Phase 5 — 複数 Consumer 検証

8. `MultiConsumer same-group` — 同一 Group で分散受信
   ```powershell
   & $mvn exec:java "-Dexec.mainClass=com.example.kafka.MultiConsumer" "-Dexec.args=pubsub-test same-group 10"
   ```
   → Consumer 間でパーティション単位に分散されること
   （実測: Consumer-1=8件(partition=2 担当), Consumer-2=10件(partition=1 担当)）

9. `MultiConsumer diff-group` — 別 Group で独立受信
   ```powershell
   & $mvn exec:java "-Dexec.mainClass=com.example.kafka.MultiConsumer" "-Dexec.args=pubsub-test diff-group 10"
   ```
   → 各 Consumer Group が独立してオフセットを管理し、新規 Group は全メッセージを受信すること
   （実測: Consumer-2(multi-group-2、新規 Group)=10件受信）
